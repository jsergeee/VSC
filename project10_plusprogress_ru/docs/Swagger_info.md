1. Исходный код Django (ваши файлы)
text
models.py → описывают структуру данных (поля, типы)
views.py → описывают эндпоинты и права доступа
serializers.py → описывают, какие поля показывать
api_urls.py → описывают URL-адреса
2. DRF (Django REST Framework) анализирует этот код
DRF смотрит на ваши ViewSet'ы, сериализаторы и модели и строит внутреннюю схему API.

3. drf-spectacular собирает всё в OpenAPI схему
text
DRF schema → OpenAPI JSON/YAML
4. Swagger UI читает эту схему и показывает
text
OpenAPI JSON → Красивый интерфейс
📊 Визуально:
text
Ваш код (models.py, views.py, serializers.py)
    ↓
DRF анализирует
    ↓
drf-spectacular создает OpenAPI схему
    ↓
JSON-файл (http://127.0.0.1:8000/api/schema/)
    ↓
Swagger UI (http://127.0.0.1:8000/api/docs/)
    ↓
То, что вы видите в браузере
🎯 Поэтому:
✅ Swagger всегда актуален — он читает ваш код напрямую

✅ Не нужно ничего писать вручную — всё генерируется автоматически

✅ Если вы меняете код — Swagger обновляется автоматически

✅ JSON-схема — это просто "слепок" вашего API на данный момент

📁 Что мы экспортируем:
